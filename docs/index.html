AI Match Context API

One endpoint. One payload. Full match intelligence.

The SportsSync AI Match Context API returns a complete, pre-joined match intelligence payload in a single call. Stop stitching together 4+ data feeds, normalizing team names, and assembling context manually.

💡 What problem does this solve? Most teams building AI sports products waste months stitching together fragmented feeds, inconsistent team names, rolling form logic, injuries, line movement, and context. This endpoint returns everything in one call.

Response Schema

Every successful response contains exactly 10 top-level sections. The shape is guaranteed — sections with no data return { "available": false } instead of being omitted.

📋 Schema Contract The v1 response contract is frozen. No fields will be removed or renamed within a major version.

match

Core match identity and scheduling.

  • id uuid — Canonical match identifier
  • external_ids object — Provider IDs (espn, odds_api, sofascore)
  • sport string — soccer, basketball, football, hockey, baseball, tennis
  • league string — Canonical league ID (e.g., eng.1, nba)
  • league_name string — Human-readable league name
  • season string — e.g., "2025-26"
  • stage string — group, regular_season, playoff, etc.
  • status string — scheduled, live, halftime, finished, etc.
  • start_time datetime — ISO 8601 UTC
  • venue object|null — name, city, country, capacity

teams

Normalized team profiles. Entity resolution is handled for you.

  • home.name / away.name string — Display name
  • home.canonical_name string — Normalized SSOT identifier
  • home.code string|null — Short code (LAL, ARS)
  • home.record object|null — wins, losses, draws, win_pct
  • home.ranking int|null — League/FIFA ranking

market

Betting market snapshot. Includes consensus, per-book lines, prediction markets, and model valuation.

  • available boolean
  • generated_at datetime|null
  • consensus object — spread, total, home_ml, away_ml, draw_ml, juice
  • books array — Per-sportsbook odds
  • prediction_markets object|null — Polymarket probabilities + volume
  • valuation object|null — fair_line, delta, has_model

form

Rolling team performance. Includes ATS, fatigue, rest days, situational context.

  • home.last_5 string — e.g., "WWDWW"
  • home.ats_last_10 number|null — Against-the-spread last 10
  • home.rest_days int|null — Days since last game
  • home.fatigue_score number|null — 0-100 fatigue indicator
  • home.situation string|null — B2B, 3IN4, Normal

injuries

Team injury reports with aggregate impact scoring.

  • home.impact_score number — 0-10 aggregate injury impact
  • home.summary string — Human-readable injury summary
  • home.players[].name string
  • home.players[].status string — out, doubtful, questionable, probable
  • home.players[].impact string — high, medium, low

metadata

Response freshness, versioning, and diagnostics.

  • api_version string — Always "v1"
  • generated_at datetime — Response generation timestamp
  • response_time_ms int — Server processing time
  • data_freshness object — Per-section freshness (fresh/stale/unavailable)
  • warnings array — Non-fatal data issues
