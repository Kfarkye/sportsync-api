Overview

SportsSync Match Context API returns pre-joined game intelligence in one response for previews, research, and automation workflows.

Every response keeps the same top-level structure. If data is unavailable, the section remains present and returns { "available": false, "reason": "..." }.

Base URL

https://api.sportsync.com/v1

Required Top-Level Sections

SectionPurpose
matchIdentity, kickoff, venue, and status
teamsHome and away team context
marketOdds, lines, and consensus view
opening_linesMarket-open lines used for line anchoring
injuriesInjury reports and impact summary
intelPregame card, picks, and line-anchor context
metadataVersion and freshness metadata

Quickstart

Start with one request and one stable parser path.

What You Get
Structured match + odds + injuries + intel
Input
match_id and API key
Output
Single JSON payload with fixed shape
cURL
curl -H "x-api-key: sk_test_••••••••••••001" \
  "https://api.sportsync.com/v1/match-context?match_id=example_match_id"

What you get back

  • match.status → game state (scheduled, in_progress, finished)
  • market.total → total line when available
  • metadata.generated_at → payload generation timestamp

Authentication

Include your API key in the x-api-key header for every request.

x-api-key: sk_live_••••••••••••••••

Use masked examples in docs and client snippets. Never embed full keys in public pages or screenshots.

Endpoint

GET /match-context?match_id=<uuid>&league=<optional>

This endpoint returns a complete pre-joined match intelligence payload for a single match ID.

Query Parameters

NameTypeRequiredNotes
